图形化操作

这里先给大家简单介绍一下，这里面的功能

  1. 🔍 查看和排序

  • 点击可以查看提交历史

  • 可以按时间、作者等排序提交记录

  • 在这里可以看到完整的 Git 提交树结构

  1. 🌳 回溯树结构查看

  • 在”查看和排序”中打开提交历史

  • 可以看到所有提交的图形化树状结构

  • 点击任意提交可以查看详细信息

  1. 📥 拉取

  • 从远程仓库获取最新代码

  • 会自动合并到当前分支

  • 相当于 git pull

  1. 📤 推送

  • 将本地提交推送到远程仓库

  • 相当于 git push

  1. 📋 克隆

  • 克隆远程仓库到本地

  • 相当于 git clone

  1. 🔀 签出到…

  • 切换到其他分支或提交

  • 可以查看历史提交并签出到任意版本

  • 相当于 git checkout 或 git switch

  1. 🔄 抓取

  • 从远程获取最新代码但不自动合并

  • 相当于 git fetch

  • 之后可以手动选择合并

  1. ✅ 提交更改

  • 提交暂存区的更改

  • 相当于 git commit

  1. 🏷️ 分支/远程/存储/标记

  • 分支 ：查看、创建、删除分支

  • 远程 ：查看远程仓库信息

  • 存储 ：临时保存未提交的更改

  • 标记 ：查看 Git 标签

  1. 💻 显示 GIT 输出

  • 显示 Git 命令的详细输出

  • 用于调试和查看详细操作日志

项目代码推送流程

我们一般拿到一个项目先把项目 pull 下来，也就是拉取项目，随后创建一个新的分支，对这个分支进行操作，操作完并测试完全了，再次 pull 主分支代码（为了保证代码是最新的，防止未更新代码），切换回主分支，把我们新创建的分支进行合并并 push 上去代码即可。
