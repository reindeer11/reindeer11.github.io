git操作流程（代码和图形化操作）

2026-02-17

本文将全面解析 Git 的基本操作，从代码层面的详细解析到图形化工具的使用方法，涵盖项目从本地开发到 GitHub 远程仓库的完整推送流程。内容结合实际开发场景，完整展现一个标准开发周期中 Git 的使用规范与操作流程，帮助开发者建立规范的版本控制意识。文中分享的经验技巧均基于个人实践，仅代表个人见解，仅供参考，欢迎交流探讨。

# 基本操作
git init
git config user.name "名字"        # 设置用户名
git config user.email "邮箱"       # 设置邮箱
git status #查看当前状态
git remote add origin git@github.com:xxx/你的项目名.git# 添加SSH方式的远程仓库
git remote set-url origin git@github.com:reindeer11/你的项目名.git# 或者修改现有的远程仓库为SSH
git remote -v #查看远程仓库
git checkout -b 新分支名 #创建分支并切换
git push -u origin main# 首次推送,这其实设置了上游分支也就是main 完整语法git push -u origin 本地分支：远程分支后续可以直接git push
git add .                         # 添加到暂存区
git commit -m "提交信息"           # 提交
git branch #查看当前分支状态
git push
  • git remote add origin git@github.com:xxx/你的项目名.git# 添加 SSH 方式的远程仓库

（1）会返回俩个值

origin git@github.com:reindeer11/testdemo.git (fetch) 这个是指的是从哪来 clone 代码

origin git@github.com:reindeer11/testdemo.git (push)这个是 push 代码到哪

如何修改？什么时候要修改？

git remote set-url origin git@github.com:用户名/项目.git

git remote set-url --push origin git@github.com:另一个用户名/项目.git

日常操作查看

Git 基本操作速查

初始化和配置

git init                          # 初始化仓库
git config user.name "名字"        # 设置用户名
git config user.email "邮箱"       # 设置邮箱

日常三步

git add .                         # 添加到暂存区
git commit -m "提交信息"           # 提交
git push                          # 推送

分支操作

git branch                        # 查看分支
git checkout -b 新分支名           # 创建并切换分支
git checkout 分支名                # 切换分支
git merge 分支名                   # 合并分支

远程仓库

git remote add origin URL         # 添加远程仓库
git remote -v                     # 查看远程仓库
git clone URL                     # 克隆仓库
git pull                          # 拉取代码
git push -u origin main           # 首次推送

撤销操作

git reset                         # 清空暂存区
git checkout -- 文件名             # 撤销修改

查看状态

git status                        # 查看状态
git log                           # 查看历史

删除远程仓库文件操作

# 删除文件
git rm -r 文件夹名
# 删除文件夹
git rm --cached -r .trae

实际上删除 git 的内容，但是本地文件没有输出，然后在 push 上去。

其他操作

# 清除暂存区内容
git rm --cached 文件名#清除单个
git rm --cached -r . #清除所有
git restore --staged . #撤回最后一次的add
git reset #完全重置慎用

回溯操作

git reflog #查看当前提交的树结构，head中会存放所有操作可以直接进行回溯
git reset --hard #回溯编号

图形化操作

这里先给大家简单介绍一下，这里面的功能

  1. 🔍 查看和排序

  • 点击可以查看提交历史

  • 可以按时间、作者等排序提交记录

  • 在这里可以看到完整的 Git 提交树结构

  1. 🌳 回溯树结构查看

  • 在”查看和排序”中打开提交历史

  • 可以看到所有提交的图形化树状结构

  • 点击任意提交可以查看详细信息

  1. 📥 拉取

  • 从远程仓库获取最新代码

  • 会自动合并到当前分支

  • 相当于 git pull

  1. 📤 推送

  • 将本地提交推送到远程仓库

  • 相当于 git push

  1. 📋 克隆

  • 克隆远程仓库到本地

  • 相当于 git clone

  1. 🔀 签出到…

  • 切换到其他分支或提交

  • 可以查看历史提交并签出到任意版本

  • 相当于 git checkout 或 git switch

  1. 🔄 抓取

  • 从远程获取最新代码但不自动合并

  • 相当于 git fetch

  • 之后可以手动选择合并

  1. ✅ 提交更改

  • 提交暂存区的更改

  • 相当于 git commit

  1. 🏷️ 分支/远程/存储/标记

  • 分支 ：查看、创建、删除分支

  • 远程 ：查看远程仓库信息

  • 存储 ：临时保存未提交的更改

  • 标记 ：查看 Git 标签

  1. 💻 显示 GIT 输出

  • 显示 Git 命令的详细输出

  • 用于调试和查看详细操作日志

项目代码推送流程

我们一般拿到一个项目先把项目 pull 下来，也就是拉取项目，随后创建一个新的分支，对这个分支进行操作，操作完并测试完全了，再次 pull 主分支代码（为了保证代码是最新的，防止未更新代码），切换回主分支，把我们新创建的分支进行合并并 push 上去代码即可。

## Git分支命名规范
### 1. 通用命名规则
✅ 推荐做法：

- 使用小写字母
- 使用连字符 - 或斜杠 / 分隔
- 使用有意义的名称
- 保持简洁但描述性强
❌ 避免：

- 使用大写字母
- 使用空格
- 使用特殊字符（除了 - 和 / ）
- 过于简短或模糊的名称
